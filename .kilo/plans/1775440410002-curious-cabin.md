# Plan: Update Super Agent CLI with New Providers

## Overview

Add three new LLM providers (Kilo Gateway, OpenCode Zen, Cline) with dynamic model fetching, update TUI to display pricing/context windows, unify /models and /providers selection, and add interactive API key configuration.

---

## Provider API Configuration

| Provider     | Base URL                          | Notes            |
| ------------ | --------------------------------- | ---------------- |
| Kilo Gateway | `https://api.kilo.ai/api/gateway` | Requires API key |
| OpenCode Zen | `https://opencode.ai/zen/v1/`     | Requires API key |
| Cline        | `https://api.cline.bot/api/v1`    | Requires API key |

### Pricing Source

- Fetch pricing data from: `https://models.dev/api.json`

---

## Phase 1: Add New Provider Implementations

### 1.1 Update Settings Manager

Update `src/utils/settings-manager.ts` to add new providers:

- Add to `PROVIDER_MODELS` (empty arrays for dynamic fetch)
- Add to `DEFAULT_USER_SETTINGS.providers` with base URLs

### 1.2 Register Providers in SuperAgent

Update `src/agent/super-agent.ts` to instantiate new providers using OpenAICompatibleProvider.

---

## Phase 2: Pricing Data Fetching

### 2.1 Add Pricing to Settings Manager

In `src/utils/settings-manager.ts`:

- Add `pricing_api` config to UserSettings
- Add `ModelMetadata` interface
- Add methods:
  - `fetchPricingData(forceRefresh?: boolean): Promise<ModelMetadata[]>`
  - `getModelMetadata(modelId: string, providerId: string): ModelMetadata | undefined`
  - `getProviderModelsWithMetadata(providerId: string): ModelMetadata[]`

- Fetch from: `https://models.dev/api.json`
- Cache locally in `~/.super-agent/pricing-cache.json`

---

## Phase 3: TUI Updates

### 3.1 Update ProviderSelection Component

File: `src/ui/components/provider-selection.tsx`

- Add pricing info display
- Show API key status indicator

### 3.2 Update ModelSelection Component

File: `src/ui/components/model-selection.tsx`

- Add columns for: Pricing (input/output), Context Window
- Format: `$X.XM/$X.XM | XXXk ctx`

### 3.3 Unified Selection with Hotkeys

File: `src/hooks/use-input-handler.ts`

**Unified picker state:**

- Single state controls both provider and model selection
- Two sections: Providers and Models

**Hotkey-based switching:**

- `Shift+P` - Switch to providers view
- `Shift+M` - Switch to models view

**Selection flow:**

```
/models or /provider -> Shows unified picker
├── Providers section (providers list)
│   ├── kilo-gateway [active] ★
│   ├── opencode-zen
│   ├── cline
│   └── ...existing providers
└── Models section (models for active provider)
    ├── model-1
    └── model-2

Navigation: ↑↓ arrows, Enter select, Esc cancel
Hotkeys: Shift+P providers, Shift+M models
```

## Phase 4: Interactive API Key Configuration

### 4.1 Add /provider config Command

File: `src/hooks/use-input-handler.ts`

Add interactive configuration flow for `/provider config` command:

- Prompt for provider selection (if not specified)
- Show current configuration
- Allow setting:
  - API key (masked input)
  - Base URL (optional override)
  - Model selection
  - Account ID (for workers-ai)

**Implementation approach:**

- Use existing confirmation service pattern
- Show current values as defaults
- Allow blank to keep existing value
- Validate API key is set before allowing activation

## Key Details

- **Dynamic fetch only** - No static model lists, rely on API
- **Pricing from models.dev** - Fetch and cache from external API
- **Hotkey navigation** - Shift+P for providers, Shift+M for models
- **Interactive config** - /provider config opens interactive setup flow
