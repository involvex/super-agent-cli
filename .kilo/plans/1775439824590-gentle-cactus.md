# Plan: Update Super Agent CLI with New Providers

## Overview

Add three new LLM providers (Kilo Gateway, OpenCode Zen, Cline) with dynamic model fetching, update TUI to display pricing/context windows, and unify /models and /providers selection.

---

## Provider API Configuration

| Provider     | Base URL                          | Notes            |
| ------------ | --------------------------------- | ---------------- |
| Kilo Gateway | `https://api.kilo.ai/api/gateway` | Requires API key |
| OpenCode Zen | `https://opencode.ai/zen/v1/`     | Requires API key |
| Cline        | `https://api.cline.bot/api/v1`    | Requires API key |

### Pricing Source

- Fetch pricing data from: `https://models.dev/api.json?sort=input-costper&order=asc`

---

## Phase 2: Model Metadata Structure

### 2.1 Create Model Metadata Type

Add new type file `src/types/models.ts`:

```typescript
export interface ModelInfo {
  id: string;
  name: string;
  provider: string;
  pricing?: {
    input: number; // per 1M tokens
    output: number; // per 1M tokens
  };
  contextWindow: number; // tokens
  maxOutputTokens?: number;
  capabilities?: string[]; // ["chat", "tools", "vision", "thinking"]
}
```

### 2.2 Update Provider Interface

Modify `src/core/llm-provider.ts` to add method:

```typescript
listModelDetails(): Promise<ModelInfo[]>;
```

---

## Phase 3: Pricing Data Fetching

### 3.1 Create Pricing Service

Create `src/utils/pricing-service.ts`:

- Fetch from `https://models.dev/api.json?sort=input-costper&order=asc`
- Cache results locally
- Map model IDs to pricing info
- Provide fallback for unknown models

## Phase 4: TUI Updates

### 4.1 Update ProviderSelection Component

File: `src/ui/components/provider-selection.tsx`

- Add columns for: Provider, Model Count, Status
- Show API key status indicator

### 4.2 Update ModelSelection Component

File: `src/ui/components/model-selection.tsx`

- Add columns for: Model Name, Pricing (input/output), Context Window, Capabilities
- Format: `$X.XM/$X.XM | XXXk ctx | tags`

### 4.3 Unified Selection with Hotkeys

File: `src/hooks/use-input-handler.ts`

**Hotkey-based switching:**

- `Shift+P` - Switch to providers view
- `Shift+M` - Switch to models view
- Both `/models` and `/providers` commands open unified picker
- Show providers section and models section
- Active section highlighted

**Unified selection flow:**

```
/models or /providers -> Shows unified list
├── Providers section (press Shift+P or arrow to navigate)
│   ├── kilo-gateway [active]
│   ├── opencode-zen
│   ├── cline
│   └── ...existing providers
└── Models section (press Shift+M or arrow to navigate)
    ├── model-1
    └── model-2
```
